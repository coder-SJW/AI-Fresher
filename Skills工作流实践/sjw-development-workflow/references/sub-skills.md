# 子 Skills 参考文档

本文档描述了 sjw-development-workflow 协调的三个子 skills。

## 1. sjw-requirement-analysis（需求分析）

**触发条件：**
- 用户说"分析需求"、"拆解需求"、"生成需求文档"

**输入：**
- 用户需求描述（文本或口头描述）

**输出：**
- 需求分析文档（Markdown）
  - 背景与目标
  - 功能需求拆解
  - 非功能性需求
  - 技术实现指导
  - 风险和考虑

**质量检查点：**
- [ ] 业务目标明确
- [ ] 功能需求拆解完整
- [ ] 非功能性需求已识别
- [ ] 技术考虑合理

**调用方式：**
```
用户：[需求描述]
AI：触发 sjw-requirement-analysis skill
```

---

## 2. sjw-feature-design（功能设计）

**触发条件：**
- 用户说"进行功能设计"、"生成设计文档"、"基于需求文档设计"
- 需求分析文档完成后自动触发（在全自动模式下）

**输入：**
- 需求分析文档
- 项目技术栈和上下文

**输出：**
- 功能设计文档（Markdown）
  - 设计概述
  - 功能模块设计
  - 数据模型设计
  - API 接口设计
  - 业务流程设计
  - 非功能性设计
  - 技术实现要点
  - 测试策略
  - 部署方案

**质量检查点：**
- [ ] 模块划分清晰
- [ ] 数据模型完整
- [ ] API 接口定义完整
- [ ] 业务流程设计合理
- [ ] 考虑了性能和安全

**调用方式：**
```
用户：基于上面的需求文档进行功能设计
AI：触发 sjw-feature-design skill
```

---

## 3. sjw-feature-code（编码实现）

**触发条件：**
- 用户说"生成代码"、"实现这个功能"、"编写后端代码"
- 功能设计文档完成后自动触发（在全自动模式下）

**输入：**
- 功能设计文档
- 现有项目代码结构

**输出：**
- 完整的 Spring Boot 后端代码
  - Entity 实体类
  - Repository 接口
  - Service 业务逻辑类
  - Controller 控制器
  - DTO 类（Request/Response）
  - Mapper 对象映射
  - 数据库迁移脚本
  - 完整的测试代码
  - 影响范围分析文档

**质量检查点：**
- [ ] 代码遵循 Spring Boot 最佳实践
- [ ] 代码结构清晰
- [ ] 测试覆盖率达标
- [ ] 异常处理完整
- [ ] 事务管理正确

**调用方式：**
```
用户：基于设计文档生成完整的 Spring Boot 代码
AI：触发 sjw-feature-code skill
```

## 子 Skills 之间的依赖关系

```
用户需求
    ↓
sjw-requirement-analysis
    ↓ (需求分析文档)
sjw-feature-design
    ↓ (功能设计文档)
sjw-feature-code
    ↓ (可运行代码)
部署上线
```

## 协调策略

**全自动模式：**
- 依次调用三个 skills
- 每个步骤完成后自动进入下一步
- 遇到问题暂停，等待用户解决后继续

**分步模式：**
- 一次只执行一个 skill
- 每步完成后等待用户确认
- 用户可以调整上一步的输出
- 用户可以跳过某些步骤

**混合模式：**
- 某些步骤自动执行
- 关键步骤需要用户确认
- 根据项目情况灵活调整
