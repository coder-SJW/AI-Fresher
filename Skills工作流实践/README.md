# SJW 开发技能体系

完整的从需求到代码的自动化开发工作流。

## 🎯 概述

本技能体系提供了从用户需求到可运行代码的完整自动化流程，包含四个核心 skills：

1. **需求分析** (sjw-requirement-analysis) - 将用户需求转化为结构化的需求文档
2. **功能设计** (sjw-feature-design) - 基于需求文档生成详细的功能设计
3. **编码实现** (sjw-feature-code) - 基于设计文档生成可运行的 Spring Boot 代码
4. **工作流编排** (sjw-development-workflow) - 协调整个开发流程，提供项目管理、质量控制和自我进化能力

## 🚀 快速开始

### 方式 1：使用编排器（推荐）

```
用户：开始完整开发流程：[需求描述]
→ sjw-development-workflow 自动协调整个流程
→ 完成后输出完整的项目文档和代码
```

### 方式 2：分步执行

```
# 步骤 1：需求分析
用户：帮我分析这个需求：[需求描述]
→ sjw-requirement-analysis 生成需求文档

# 步骤 2：功能设计
用户：基于需求文档进行功能设计
→ sjw-feature-design 生成设计文档

# 步骤 3：编码实现
用户：基于设计文档生成代码
→ sjw-feature-code 生成完整代码
```

## 📋 执行模式

### 全自动模式

- ✅ 一次性完成整个流程
- ✅ 适合需求明确的场景
- ⚠️ 质量问题时会暂停

### 分步执行模式

- ✅ 每步都需要用户确认
- ✅ 适合复杂项目
- ✅ 可以随时调整

### 混合模式

- ✅ 平衡效率和灵活性
- ✅ 关键步骤需要确认
- ✅ 适合大多数场景

## 🔧 核心功能

### 1. 需求分析

**输入：** 用户需求描述
**输出：** 需求分析文档
**包含：**
- 背景与目标
- 功能需求拆解（史诗、功能、用户故事）
- 非功能性需求
- 技术实现指导
- 风险和考虑

### 2. 功能设计

**输入：** 需求分析文档
**输出：** 功能设计文档
**包含：**
- 功能模块设计
- 数据模型设计（ERD、表结构）
- API 接口设计（RESTful）
- 业务流程设计（流程图、状态机）
- 非功能性设计
- 技术实现要点

### 3. 编码实现

**输入：** 功能设计文档
**输出：** 完整的 Spring Boot 后端代码
**包含：**
- Entity 实体类
- Repository 接口
- Service 业务逻辑
- Controller 控制器
- DTO 类（Request/Response）
- Mapper 对象映射
- 数据库迁移脚本
- 完整的测试代码
- 影响范围分析

### 4. 工作流编排

**功能：**
- 🔄 协调三个子 skills 的执行
- 📊 项目进度跟踪
- ✅ 质量门禁检查
- 📁 文档管理
- 🔄 迭代优化建议
- 🧬 自我进化（优化子 skills）

## 🎯 使用场景

### 场景 1：快速原型开发

```
用户：开发一个用户管理功能的原型，全自动模式
→ 30分钟内完成需求、设计、代码
→ 输出完整的 Spring Boot 后端代码
```

### 场景 2：正式项目开发

```
用户：开始分步开发流程
→ 需求分析完成后，与团队评审
→ 评审通过后进入功能设计
→ 设计确认后生成代码
→ 每步都有质量把关
```

### 场景 3：增量功能开发

```
用户：在现有项目上添加新功能
→ 自动分析现有代码结构
→ 遵循项目规范生成代码
→ 提供影响范围分析
```
